# daily-mood-checkin のフォルダ・ファイル一覧（やさしく）

このプロジェクトにある**フォルダとファイル**を、役割ごとにやさしく説明します。  
「何に使うか」「触っていいか」がわかるようにしています。

---

## 一覧の見方

- **触る** … 中身を変えたり追加したりするファイル
- **読む** … 説明や記録を読むためのドキュメント
- **実行する** … ダブルクリックやコマンドで動かすもの
- **自動** … プログラムが自動で作る・更新する。普段は触らない

---

## フォルダの上（プロジェクトの直下）

```
daily-mood-checkin/

├── README.md
├── Dockerfile
├── .dockerignore
├── .gitignore
├── サーバーを起動.command

├── docs/            … フォルダ
```

## ファイルの説明（直下）

| 名前 | 役割（やさしく） | 触る？ |
|------|------------------|--------|
| **app.py** | バックエンドの本体。API（今日の気分を返す・保存する・一覧を返す）と、DB（mood.db）への保存処理が書いてある。サーバーを起動するとこのファイルが動く。 | 触る（機能を変えるとき） |
| **requirements.txt** | 「このアプリを動かすのに必要な Python のライブラリ一覧」。中身はいまは Flask だけ。`pip install -r requirements.txt` で一括インストールするときに読まれる。 | 触る（ライブラリを増やすとき） |
| **README.md** | プロジェクトの説明と、動かし方・ログイン時に開く設定・うまく動かないときの対処が書いてある。最初に読むとよい。 | 読む |
| **Dockerfile** | Docker で動かすときの「設計図」。中で「Python を入れて、app.py と static をコピーして、5001番で起動する」と書いてある。Docker を使わないなら見なくてよい。 | 読む／触る（Docker を使うとき） |
| **.dockerignore** | Docker でイメージを作るときに「このファイル・フォルダはコピーしない」と指定する一覧。mood.db や .venv などを除いて、イメージを軽くするため。 | 普段は触らない |
| **.gitignore** | Git で管理するときに「このファイル・フォルダはコミットしない」と指定する一覧。mood.db（記録）や .venv（環境）などを除外して、リポジトリをすっきりさせるため。 | 普段は触らない |
| **サーバーを起動.command** | ダブルクリックすると、このフォルダに移動してから Python で app.py を起動する。サーバーが止まったときにいちばん手軽に起動する方法。 | 実行する（ダブルクリック） |
| **mood.db** | SQLite のデータベースファイル。気分の記録（日付と1〜5の点数）が入っている。アプリを初めて動かしたときに自動で作られ、その後も自動で更新される。 | 普段は触らない（バックアップしたいときはコピー） |

## static フォルダ（画面まわり）

**役割** … ブラウザに表示する「画面」の部品が入っている。HTML（骨組み）・CSS（見た目）・JS（動き・API呼び出し）。

| 名前 | 役割（やさしく） | 触る？ |
|------|------------------|--------|
| **index.html** | トップページの骨組み。「ここにタイトル」「ここに今日の気分」「ここにグラフ」「ここに1〜5のボタン」と並びを決めている。 | 触る（文言や並びを変えるとき） |
| **style.css** | 見た目の指定。色・余白・フォント・ボタンの形・グラフのスタイルなど。 | 触る（デザインを変えるとき） |
| **app.js** | 画面の動きとサーバーとのやりとり。ページを開いたときに今日の気分を取得、1〜5を押したときに保存、グラフや一覧を描く。 | 触る（動きや表示を変えるとき） |

## scripts フォルダ（起動・自動化まわり）

**役割** … Mac で「ログインしたらサーバーを起動してブラウザを開く」ための設定や、そのときに使うスクリプト。

| 名前 | 役割（やさしく） | 触る？ |
|------|------------------|--------|
| **start-mood-checkin.sh** | プロジェクトに移動 → Python で app.py を起動 → 少し待ってからブラウザで http://localhost:5001 を開く、という処理が書いてある。Launch Agent から呼ばれる。 | 普段は触らない（パスを変えるときだけ） |
| **com.moodcheckin.plist** | Launch Agent 用の設定。「ログインしたら start-mood-checkin.sh を実行する」と Mac に伝える。これを ~/Library/LaunchAgents/ にコピーして load すると、ログイン時に自動起動する。 | 読む／触る（設定を変えるとき） |
| **launchd.log** | Launch Agent がスクリプトを実行したときの「標準出力」のログ。ある場合のみ。 | 読む（うまく動かないときの確認） |
| **launchd.err.log** | Launch Agent がスクリプトを実行したときの「エラー出力」のログ。ある場合のみ。 | 読む（うまく動かないときの確認） |

## .venv フォルダ（Python の仮想環境）

**役割** … このプロジェクト専用の Python 環境。Flask など必要なライブラリがここにインストールされている。`pip install -r requirements.txt` で作ったときに自動でできる。

**普段は中身を触らない。** 削除すると、もう一度 `python3 -m venv .venv` と `pip install -r requirements.txt` で作り直す必要がある。

| 記録のデータをバックアップしたい | **mood.db** をコピー |
