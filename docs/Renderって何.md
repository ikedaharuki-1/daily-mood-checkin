# Render って何？

**Render** は、あなたのプログラム（アプリ）を **インターネット上で動かして、URL をくれるサービス** です。

---

## 一言でいうと

**「あなたの代わりにサーバーを用意して、そこであなたのアプリを動かしてくれる会社」** です。

- あなたがやること：GitHub にコードを置く → Render に「このリポジトリを動かして」と設定する
- Render がやること：コードを取ってきて、自分たちのコンピュータでビルド・起動し、**https://〇〇.onrender.com** のような URL をくれる

だから「サーバーを立てる」の「サーバー」を、**Render が用意している**イメージです。

---

## ローカルと比べると

|  | ローカル（あなたの Mac） | Render |
|--|--------------------------|--------|
| **サーバーはどこ？** | あなたの PC のなか | **Render のコンピュータ**（クラウド） |
| **誰が起動する？** | あなた（「サーバーを起動.command」など） | **Render**（デプロイすると自動で起動） |
| **URL** | http://localhost:5001（自分のPCだけ） | **https://daily-mood-checkin.onrender.com**（どこからでも） |

---

## よくある言い方

- **PaaS（パース）** … 「Platform as a Service」の略。アプリを動かすための「場所と仕組み」をサービスとして提供する、という意味。Render はこの一種。
- **クラウド** … インターネットの向こう側にあるコンピュータで動かすこと。Render はクラウドの上で動いている。

---

## 今回あなたがしたこと

1. GitHub に **daily-mood-checkin** のコードを置いた  
2. Render で「**このリポジトリを Web サービスとして動かして**」と設定した（Build / Start コマンドを指定）  
3. Render が **自分のサーバーで** あなたのアプリをビルド・起動し、**URL を発行**した  

＝ **Render ＝ あなたのアプリを「公開用サーバー」で動かしてくれるサービス**、という理解で大丈夫です。
