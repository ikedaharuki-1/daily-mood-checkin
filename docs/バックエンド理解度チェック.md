# 答えと簡単な解説

---

## 第1問の答え

**3. `post_mood()`**

- 「保存する」＝サーバーに**送る（POST）**ので、`post_mood()` が担当します。  
- `get_today()` は「今日の気分を**取得**」、`get_moods()` は「一覧を**取得**」です。

---

## 第2問の答え

**正しいもの：A と C**

- **A** … 正しい。mood.db には気分の記録が入っています。  
- **B** … 誤り。mood.db は**サーバー（あなたのPC）側**にあり、ブラウザのなかにはありません。  
- **C** … 正しい。moods 表の列は id, day, score で、日付と点数が入ります。  
- **D** … 誤り。初めて API を動かしたときに **init_db()** が表を作り、SQLite が mood.db ファイルを自動で作ります。

---

## 第3問の答え

**4. `GET /api/moods`**

- 「一覧を**取得**」なので **GET**。  
- 一覧用の窓口は **/api/moods** です。  
- `/api/today` は「今日の1件だけ」、`/api/mood` は「保存する（POST）」のときに使います。

---

## 第4問の答え

**正しい順序：イ → エ → ア → ウ**

1. **イ** … リクエストを受け取る（`request.get_json()` で score: 5 を読む）  
2. **エ** … 1〜5 かチェックする  
3. **ア** … 今日の日付と 5 を DB に保存する（`INSERT OR REPLACE`）  
4. **ウ** … 「ok: true」を返す（`return jsonify(...)`）

---

## 第5問の答え

- **(1) `get_db`** … mood.db に接続して、その接続（conn）を返す関数です。  
- **(2) `init_db`** … moods 表がなければ CREATE TABLE で作る関数です。  
- **(3) `POST /api/mood`** … 気分を保存する窓口は、メソッドが POST、URL が /api/mood です。

---

## 第6問の答え

**「moods 表のなかから、day が（今日の日付）の行だけを取り出し、その行の day と score を取得する」**

- SELECT … 取り出す列（day, score）  
- FROM moods … どの表か（moods）  
- WHERE day = ? … どの行か（? に今日の日付が入る）

＝「今日の気分が記録されているか、されていればその点数」を取るための SQL です。

---

## 第7問の答え（例）

**「URL とメソッド（例：GET /api/today）が窓口で、その窓口に来たときに実行される関数（get_today() など）が実際の処理」**

- `@app.route("/api/today", methods=["GET"])` が「この URL が窓口」と宣言し、  
- その下の `def get_today():` が「窓口に来たときに動く処理」です。
