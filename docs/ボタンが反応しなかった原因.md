# ボタンが反応しなかった原因と対応

## 根本原因

**HTML から読み込んでいた `app.js` と `style.css` の URL が間違っており、サーバーから 404 になっていました。**

- ページは `http://localhost:5001/` で配信されている
- HTML には `<script src="app.js">` と `<link href="style.css">` とだけ書いてあった
- ブラウザは相対URLなので `http://localhost:5001/app.js` と `http://localhost:5001/style.css` をリクエストする
- Flask では静的ファイルは **`/static/`** の下だけ配信するため、**`/app.js` や `/style.css` にはルートがなく 404**
- その結果 **JavaScript が一切読み込まれず、クリック処理が動いていなかった**

## 実施した修正

1. **静的ファイルのパスを修正**
   - `app.js` → **`static/app.js`**
   - `style.css` → **`static/style.css`**
   - これで `http://localhost:5001/static/app.js` などが正しく返るようになった。

2. **クリックまわりを強化**
   - `document.body` にイベントを張る**イベント委任**に変更（クリックを確実に受け取る）
   - 押した瞬間にモーダルを閉じて「送信中…」と表示するようにした
   - 要素取得を `getElementById` でその都度行い、null チェックを入れてエラーで止まらないようにした

3. **キャッシュ対策**
   - スクリプトに `?v=5` を付与
   - 開発時に静的ファイルでキャッシュされないよう `Cache-Control` を付与

## 確認方法

1. サーバーを起動する（「サーバーを起動.command」または `python app.py`）
2. ブラウザで **http://localhost:5001** を開く（キャッシュを無視して再読み込み: Ctrl+Shift+R / Cmd+Shift+R 推奨）
3. 1〜5 のいずれかを押す → モーダルが閉じ、「今日の気分: ○ / 5（記録済み）」と表示されればOK
