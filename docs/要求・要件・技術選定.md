# 朝イチ気分チェック — 要求・要件・技術選定（ライト版）

## 1. 要求定義（なぜ・誰が・何をしたいか）

| 項目 | メモ |
|------|------|
| **背景・課題** | 日々の気分の変化を軽く記録したい。日常で使えるものにしたい。 |
| **主な利用者** | 自分ひとり |
| **やりたいこと** | 朝、PCを開いたとき（＝その日はじめてページを開いたとき）に、5段階で気分を選んで入力したい |
| **成功のイメージ** | 毎朝ポップアップで選ぶだけ。記録が残り、あとで振り返れる。 |

---

## 2. 要件定義（何ができればよいか）

### 必須（MVP）

- [x] その日1回だけ、5段階で気分を入力できる
- [x] 入力したら「今日は記録済み」となり、同じ日はポップアップが出ない
- [x] データは保存され、一覧で確認できる

### あるとよい（余裕があれば）

- [x] 過去の気分の一覧・簡易グラフ
- [x] ログイン時にブラウザでこのページを開く設定の案内（README等）

### しないこと（スコープ外）

- 複数ユーザー、ログイン認証、スマホアプリ

---

## 3. 技術選定（何で作るか）

| 役割 | 候補 | 選ぶ理由（ライトに） |
|------|------|----------------------|
| **バックエンド** | Python + Flask | 1ファイルでAPIが書ける。文法が読みやすく習得しやすい。 |
| **データ保存** | SQLite | ファイル1つでDB。サーバー不要・設定不要。 |
| **フロント** | HTML + CSS + 素のJS | ビルド不要ですぐ動かせる。フレームワークなしで理解しやすい。 |
| **インフラ** | Docker | 1コマンドで起動。本番に近い体験でインフラの理解が深まる。 |
| **デプロイ** | まずはローカルのみ | 1日で完結させる。必要になったらクラウド等を検討。 |
