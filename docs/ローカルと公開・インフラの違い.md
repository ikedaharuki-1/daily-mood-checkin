# ローカルと公開 — そして「インフラ」を作るとは

「サーバーを立てる」を**自分のPCの外**でやり、**インフラ**の理解を深めるための説明です。

---

## 1. ローカル vs 公開（デプロイ）

|  | ローカル（いままで） | 公開（これからやること） |
|--|----------------------|---------------------------|
| **サーバーが動く場所** | あなたの Mac のなか | **クラウドのコンピュータ**（Render などが用意している） |
| **URL** | http://localhost:5001（自分のPCだけ） | **https://〇〇.onrender.com** のような**誰でもアクセスできるURL** |
| **止めるのは誰？** | あなた（ウィンドウを閉じるなど） | サービス側（一定時間使わないとスリープする場合もある） |
| **データ（mood.db）** | あなたのPCのフォルダに保存 | クラウドの「そのサーバー」のなかに保存（無料枠では再起動で消える場合あり） |

**まとめ**  
- **ローカル** ＝ 自分のPCのなかだけで動かす。外からは見られない。  
- **公開（デプロイ）** ＝ クラウドの「誰かのコンピュータ」の上で動かす。URL を教えればほかの人も開ける（＝サーバーを立てた、のイメージ）。

---

## 2. 「インフラ」を作る＝どこで・どう動かすかを決めること

**インフラ** ＝ **「そのプログラムを、どこで・どんな環境で動かすか」** を用意することです。

- **ローカルのときのインフラ**  
  「あなたの Mac のなかで、Python で app.py を動かす」  
  → 用意したもの：起動用スクリプト（サーバーを起動.command）や Launch Agent。

- **公開するときのインフラ**  
  「Render のサーバーで、Flask アプリを動かす」  
  → 用意するもの：  
  - **どこで動かすか** … Render に「Web サービス」を1つ作る  
  - **どう動かすか** … コードを GitHub から渡して、**ビルド**（必要なものを入れる）と**起動コマンド**（gunicorn で app を動かす）を指定する  

つまり「**インフラを作る**」＝ **「サーバー（実行場所）と、そこで動かす手順を用意する」** ことです。

---

## 3. 今回やることの流れ（図）

```
  いま（ローカル）
  あなたの Mac ──► ブラウザで localhost:5001 を開く
       │
       └── app.py と mood.db が同じPCのなか

  これから（公開）
  GitHub にコードを置く
       │
       ▼
  Render が「そのコードを取って、サーバーで動かす」
       │
       ▼
  あなたや誰か ──► ブラウザで https://〇〇.onrender.com を開く
       │
       └── app.py は Render のコンピュータで動く
           mood.db もそのコンピュータのなか（無料枠では再起動で消える場合あり）
```

**やること**  
1. コードを GitHub に置く（まだならリポジトリを作って push）  
2. Render で「Web サービス」を1つ作り、「この GitHub リポジトリからデプロイする」と設定する  
3. ビルド・起動のコマンドを指定する（後述の手順のとおり）  
4. デプロイが終わると **URL がもらえる** → それが「立てたサーバー」の住所になる  

---

## 4. 用語の対応（このステップで出てくるもの）

| 用語 | やさしく |
|------|----------|
| **デプロイ** | プログラムを「本番用の場所」に置いて、そこで動くようにすること。ここでは「GitHub のコードを Render に渡して動かす」までを指す。 |
| **ビルド** | デプロイ先の環境で「必要なものを入れる」こと。ここでは `pip install -r requirements.txt` で Flask や gunicorn を入れる。 |
| **起動コマンド** | サーバーで「このコマンドでアプリを起動してね」と指定するもの。ここでは `gunicorn` で app を動かす。 |
| **gunicorn** | 本番向けの Web サーバー。Flask の開発サーバーより安定して動かすためのもの。Render ではこれで app を動かす。 |
| **環境変数** | プログラムに外から渡す「設定の値」。例：`PORT`（どの番号で待つか）。Render が自動で `PORT` を渡す。 |

---

## 5. 注意（無料枠で動かす場合）

- **Render の無料枠**では、しばらくアクセスがないと**スリープ**し、次のアクセスでまた起動します。起動に数十秒かかることがあります。  
- **SQLite（mood.db）**は、無料枠では「そのサーバー内の一時的な領域」に保存されるため、**サービスが再起動するとデータが消える**ことがあります。  
  - 学習目的なら「公開URLで動く」を体験するだけで十分。  
  - 本格的にデータを残したい場合は、のちに PostgreSQL などの「外付けDB」を使うやり方に変えるとよいです。
